Visual wiring guide for ESP32 + DHT22 + Water Sensor on GL No.12 breadboard

1 Place ESP32 on Breadboard

Push ESP32 into columns 1–19, straddling the center gap. USB port faces LEFT (off the board edge). The left-side pins go into Row B, right-side into Row J.

Unplug USB before wiring! All wiring must be done with power OFF.

2 Set Up Power Rails (2× M-M wires)

Connect 3.3V and GND from the ESP32 to the top power rails using M-M wires (pin on both ends).

Why M-M? ESP32 pins go DOWN into the breadboard — nothing sticks up. You reach them through Row A (the free row), which connects to the pin in Row B below.
Wire 1 Pin → Col 19, Row A (3V3) Pin → (+) INNER strip (3.3V) M-M
Wire 2 Pin → Col 6, Row A (GND) Pin → (−) OUTER strip (GND) M-M
After this, any hole on the INNER (+) strip = 3.3V, any hole on the OUTER (−) strip = GND. Your board has no colour markings — just remember: inner = power, outer = ground.
3 Wire the DHT22 Sensor (3× M-F wires)

DHT22 stays off the breadboard — just like the water sensor. It has 3 pins sticking out. Clip M-F sockets onto them.

Your DHT22 has 3 pins (same labels as water sensor!):

Wire 3 🔌 Socket on DHT22 (unlabeled pin = VCC) Pin → (+) INNER strip (3.3V) M-F
Wire 4 🔌 Socket on DHT22 (S) Pin → Col 11, Row A (GPIO 25) M-F
Wire 5 🔌 Socket on DHT22 (−) Pin → (−) OUTER strip (GND) M-F
Same method as water sensor! Socket end clips onto DHT22 pin, pin end plugs into breadboard/bus strip.
4 Wire the Water Sensor (3× M-F wires)

Water sensor also stays off the breadboard. Same method — clip M-F sockets onto pinssticking out.

Wire 6 🔌 Socket on Sensor (+) Pin → (+) INNER strip (3.3V) M-F
Wire 7 🔌 Socket on Sensor (−) Pin → (−) OUTER strip (GND) M-F
Wire 8 🔌 Socket on Sensor (S) Pin in Col 15, Row A (GPIO 34) M-F
Why M-F here? The sensor has pins sticking OUT that you can clip sockets onto. The ESP32 pins go DOWN into the breadboard — nothing to clip onto — so we use M-M through Row A instead.

☑️ Before Power-On Checklist

  • ESP32 placed: 19 pins each side, straddling center gap
  • No bent pins — all 38 pins inserted cleanly
  • Wire 1: Yellow M-M — Col 19 Row A → (+) inner strip (3.3V)
  • Wire 2: Grey M-M — Col 6 Row A → (−) outer strip (GND)
  • Wire 3: Green M-F — Socket on DHT22 (unlabeled) → Pin in (+) inner strip
  • Wire 4: Brown M-F — Socket on DHT22 (S) → Pin in Col 11 Row A (GPIO 25)
  • Wire 5: White M-F — Socket on DHT22 (−) → Pin in (−) outer strip
  • Wire 6: Red M-F — Socket on Sensor (+) → Pin in (+) inner strip
  • Wire 7: Purple M-F — Socket on Sensor (−) → Pin in (−) outer strip
  • Wire 8: Blue M-F — Socket on Sensor (S) → Pin in Col 15 Row A (GPIO 34)
  • No exposed wire ends touching
  • USB port side is accessible (not blocked by wires)

⚡ Power On

  1. Plug USB into ESP32 and PC
  2. Onboard blue LED (GPIO 2) should blink
  3. Arduino IDE → Tools → Port → select COM port
  4. Serial Monitor → 115200 baud
  5. Upload P-WOS firmware
  6. Expected output: 
    ============================================
  P-WOS ESP32 Firmware
  Commands: STATUS | READ | PUMP ON | PUMP OFF
============================================
[READY] Publishing every 5s
[DATA] Soil:XX.X%  Temp:XX.X°C  Hum:XX.X%  Pump:OFF
  7. Type STATUS to get a reading on demand

M-F socket not clipped firmlyIntermittent readingsPush socket fully onto sensor pin
Bus strip split not bridgedPower only on one halfAdd jumper across the center split
Using flash pins (SD0/SD1/CMD/CLK)ESP32 crashesAvoid cols 1–4 left, cols 1–3 right
